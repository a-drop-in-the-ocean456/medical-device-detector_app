# 医疗设备检测模型训练Pipeline
软件  vscode   anaconda
本文档描述如何使用Grounding DINO进行数据标注、训练YOLO模型、并部署到后端的完整流程。

## 第一步：数据标注

### 1.1 使用Grounding DINO自动标注

项目已提供标注工具 `标注工具/video_annotator_cli.py`，支持使用Grounding DINO对视频进行自动标注。
项目已提供标注工具 `标注工具/grounding_dino标注所有图片.py`，支持使用Grounding DINO对多张图片进行自动标注。

```bash
# 标注多张图片
python 标注工具/grounding_dino标注所有图片.py

# 标注视频（会提取帧并标注）
python 标注工具/video_annotator_cli.py
```

**关键参数说明：**
- `--textprompt`: 检测文本提示，默认"medical device"
- `--box-threshold`: 检测框阈值，默认0.3
- `--text-threshold`: 文本匹配阈值，默认0.25

### 1.2 可视化检查标注结果

使用可视化工具检查标注结果：

```bash
# 可视化所有标注
python 标注工具/visualize_yolo_labels.py --images-dir datasets/images/ --labels-dir datasets/labels/

# 可视化指定图片
python 标注工具/visualize_yolo_labels.py --images-dir datasets/images/ --labels-dir datasets/labels/ --show
```

### 1.3 标注修正

标注结果文件格式为YOLO格式（`*.txt`），每行格式为：
```
<class_id> <x_center> <y_center> <width> <height>
```
所有值为归一化坐标（0-1）。

**修正方式：**

**方式一：直接删除错误标注和对应的图片**

**方式二：使用LabelImg调整框位置**
在LabelImg中打开图片，手动调整边界框位置或删除错误的框，然后保存。

**类别ID定义（qc_dataset.yaml）：**
- 0: syringe（注射器）
- 1: Needle holder（医疗器械）
- 2: waste（医疗废物）
- 3: large gauze（大纱布）
- 4: Iodophor（碘伏）

## 第二步：模型训练

### 2.1 数据准备

确保数据集目录结构如下：
```
dataset/
├── images/
│   ├── train/      # 训练图片
│   ├── val/        # 验证图片
│   └── test/       # 测试图片
└── labels/
    ├── train/
    ├── val/
    └── test/
```

### 2.2 训练YOLOv8模型

```bash
cd ultralytics-main

# 训练模型（使用qc_dataset.yaml配置）
python train_yolov8n.py --data qc_dataset.yaml --epochs 100 --batch 16 --imgsz 640
```

**训练参数：**
- `--model`: 预训练模型（yolov8n.pt/yolov8s.pt/yolov8m.pt等）
- `--data`: 数据集配置文件路径
- `--epochs`: 训练轮数
- `--batch`: 批大小
- `--imgsz`: 输入图片尺寸

### 2.3 验证模型

```bash
# 验证模型性能
python train_yolov8n.py --model runs/detect/train/weights/best.pt --data qc_dataset.yaml --val

# 单独运行验证脚本
python ultralytics/models/rtdetr/val.py --model runs/detect/train/weights/best.pt --data qc_dataset.yaml
```

## 第三步：模型部署到后端

### 3.1 放置模型文件

将训练好的模型文件复制到后端目录：
复制best.pt到后端目录backend_new



### 3.2 后端调用

后端`detector.py`已集成YOLO检测器，默认自动加载`best.pt`：

```python
from detector import MedicalDeviceDetector

# 初始化检测器（会自动加载best.pt）
detector = MedicalDeviceDetector()

# 或指定模型路径
detector = MedicalDeviceDetector(model_path='best.pt')

# 检测图片
result = detector.detect('path/to/image.jpg')

# 返回结果
print(result['detections'])       # 检测到的物体列表
print(result['description'])    # 检测描述文本
print(result['image_base64'])    # 可视化结果图片
```

### 3.3 启动后端服务
软件使用mobaxterm
```bash
cd backend_new

# 启动服务
python app.py

# 或使用启动脚本
bash start.sh
```

后端服务启动后，通过API调用检测：
```
POST /detect
Content-Type: multipart/form-data

参数: image (图片文件)
响应: JSON (检测结果)
```

## 附录：常见问题

**Q: 标注时如何指定类别？**
A: 在`qc_dataset.yaml`中定义names，Grounding DINO会自动使用text_prompt中的类别名称。

**Q: 训练时显存不足？**
A: 减小batch size，如`--batch 8`或`--batch 4`。

**Q: 如何增加检测类别？**
A: 修改`qc_dataset.yaml`的names字段，并在标注时指定对应的text_prompt。
